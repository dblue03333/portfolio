mini-llm-twin

A production-grade RAG (Retrieval-Augmented Generation) pipeline that ingests personal knowledge, generates embeddings, and retrieves contextually relevant answers using LLMs.

System Architecture

┌──────────────────────────────────────────────────────────┐ │ mini-llm-twin Architecture │ ├──────────────────────────────────────────────────────────┤ │ │ │ [Data Source] ──→ [Chunking] ──→ [Embedding] │ │ Notion / Text Google Gemini │ │ Documents Splitter text-embedding-004 │ │ │ │ │ ▼ │ │ [MongoDB Atlas] │ │ Vector Store │ │ │ │ │ ▼ │ │ [User Query] ──→ [Retrieval] ──→ [Generation] │ │ FastAPI Vector Gemini LLM │ │ Endpoint Similarity Context + Answer │ │ │ └──────────────────────────────────────────────────────────┘

Key Features

• Feature Pipeline: Automated ingestion from data sources, intelligent text chunking, and batch embedding generation.

• Vector Search: MongoDB Atlas vector indexing for semantic similarity retrieval with cosine distance.

• RAG Generation: Context-augmented response generation using Google Gemini with prompt engineering.

• REST API: FastAPI endpoint for real-time Q&A queries with structured response format.

• Containerized: Docker-based deployment for reproducibility and portability.

Tech Stack

Python FastAPI MongoDB Atlas Google Gemini Docker Vector Search RAG Pipeline Pydantic

What I Learned

Building mini-llm-twin deepened my understanding of the full RAG lifecycle — from raw data ingestion through embedding generation to intelligent retrieval. I learned to design production-grade data pipelines, work with vector databases, implement prompt engineering strategies, and deploy AI services with Docker. This project bridges my theoretical knowledge from the NVIDIA GenAI certification with practical, hands-on engineering.
